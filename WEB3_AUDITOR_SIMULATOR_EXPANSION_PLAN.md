# Web3 Auditor 模拟器 — 玩法扩展方案（生活 / 困境 / 趣味）

> 目标：在现有 MVP（按周推进、直客/平台、行动点、事件、日志、存档）基础上，把“审计师的生活与困境”做得更像真的、更好玩：  
> **高频小决策 + 偶发大事件链**，让玩家在“赚钱/口碑/合规/身心”之间持续摇摆。

---

## 1. 核心体验：审计师到底在“困”什么
### 1.1 四座大山（驱动策略的长期张力）
- **现金流**：项目费好看但回款慢；平台奖金不稳定；工具、律师、公关都要钱。
- **声望**：你越出名，越接到烫手山芋；一波爆雷就清零。
- **合规/伦理**：代币、内幕、利益冲突、背书措辞、披露时机；“做对”往往不赚钱。
- **身心**：加班会让你短期强，但长期崩；注意力是审计师的生命值。

### 1.2 行业特色的“好玩点”
- **范围管理**比审计更难：你审计的不是代码，是“期望”。
- **漏洞发现不是终点**：写清楚、让人修、复测确认、上线后别爆雷，才算赢。
- **平台赛道**很像竞技：撞车/去重/评审/申诉/小作文，胜负常在细节和心态。
- **舆情与话语权**：你说“风险”别人听成“背书”；你说“限制”别人当作“免责”。

## 2. 建议新增的系统模块（可逐步上线）

## 2.1 “时间预算”从行动点升级为：深度工作 vs 会议 vs 文案
现有行动点很好，但可以把每周行动点进一步拆成 3 类（仍用同一个 UI，只是结算更有味儿）：
- **深度工作（Deep Work）**：审计/建模/复测（产洞、产质量）
- **沟通会议（Meetings）**：Kickoff、范围扯皮、修复跟进（保交付、保心态）
- **文案写作（Writing）**：报告、申诉材料、公告措辞（声望/通过率/纠纷风险）

> 玩法效果：同样 1 周，选“多会议”会更稳但产洞少；选“全深度工作”产洞多但可能翻车。

## 2.2 “注意力/专注”作为第二行动资源（更贴近现实）
- **精力（Stamina）**：你还能坐多久
- **专注（Focus）**：你还能看懂多少

行动消耗的主要差异：
- 审计/建模主要消耗 **Focus**，同时消耗少量 Stamina
- 会议/扯皮主要消耗 **Mood**，并压 Focus 的恢复
- 连续 2~3 周高强度会触发“脑雾/眼花/手抖”事件链，降低发现高危概率

## 2.3 “工具链/流程”改成可选装备与维护成本（有趣且真实）
把 Tooling 从单数值扩成“装备槽”（仍可简化展示为 Tooling 总分）：
- **静态分析套件**：发现低中危更稳，误报可能↑
- **Fuzz/符号执行**：成本高、对特定协议收益大
- **审计清单/模板库**：对同类型协议有常驻加成
- **内部知识库**：重复项目会越来越顺

引入维护成本：
- 工具订阅费、CI 成本、误报处理时间（让“买工具”变成策略）

## 2.4 直客交付：加入“修复率/复测门槛”
现实里最痛的是：你找到了，客户不修。
- 新增项目字段：**修复率（Fix Rate）** 与 **上线冲动（Ship Urgency）**
- 交付时如果修复率低：
  - 声望结算打折
  - “上线后爆雷事件”权重上升
- 复测动作可以把“爆雷权重”压下来，但占用你最宝贵的一周

## 2.5 平台赛道：引入“提交/冻结/申诉”三个状态
让平台更像“竞赛”而不是“自动发稿”：
- **发现（draft）**：你自己知道
- **提交（submitted）**：进入去重/评审池
- **冻结（freeze）**：临近截止，提交关闭，只能补材料
- **申诉（appeal）**：消耗写作/行动点，争取从驳回→接受或从中危→高危

关键趣味：玩家要决定“早点提交抢时间戳”还是“晚点提交写得更硬”。

## 2.6 角色生态：客户/评审/社区/黑客的“人格化”标签
给每个订单/竞赛挂 1~2 个标签，带来稳定的行为模式（让玩家有“识人”乐趣）：
- **客户标签**：压价型、爱加需求型、迷信背书型、配合修复型、跑路型
- **评审标签**：严格保守、偏工程实践、偏理论证明、爱抓措辞
- **社区标签**：理性技术党、阴阳怪气党、KOL 带节奏党
- **黑客标签**：抢跑党、MEV 老手、桥专精、供应链投毒

> 标签带来“可学习的规律”，玩家会觉得自己在成长。

## 3. 趣味事件链（建议做成“连续剧”，而非孤立事件）

## 3.1 “背书措辞”三连击（直客）
1) 客户要求背书 → 2) 公告措辞被社区误读 → 3) 若爆雷则你被追责  
分支选择示例：
- **坚持克制**：短期少拿“感谢费”，长期合规风险↓，爆雷后伤害更小
- **含糊其辞**：短期现金↑，但合规风险↑，爆雷时声望损失翻倍

## 3.2 “范围蔓延 → 加班 → 漏报”链（直客）
1) 需求增加 → 2) 你熬夜接下 → 3) 下一周 Focus 下降 → 4) 高危漏报概率↑  
玩法要点：让玩家学会“拒绝不是坏人”，而是职业能力的一部分。

## 3.3 “平台撞车 → 申诉小作文 → 逆转/破防”链（平台）
1) 去重撞车 → 2) 你决定找变体/补影响面 → 3) 评审分歧 → 4) 结算翻盘或被嘲  
玩法要点：写作与证据链的重要性（也更贴近真实平台生态）。

## 3.4 “供应链爆雷”链（全局）
1) 你审计的依赖库出现 0day → 2) 你需要发公告/建议 → 3) 客户是否采纳 → 4) 舆情/合规结算  
玩法要点：让玩家体验“不是你写的代码也会背锅”的现实。

## 3.5 “利益冲突”链（合规）
1) 有人给你代币/期权 → 2) 你接单/参赛 → 3) 评审/客户质疑利益冲突 → 4) 是否披露/是否退回  
玩法要点：合规不是“加减分”，而是“未来灾难概率”的阀门。

## 4. 更贴近“审计生活”的可选动作（让按钮更好笑也更真实）
把“动作按钮”写得像朋友圈/工位日常（仍对应同一套数值变化）：
- **“对着 PR 发呆 20 分钟”**：Focus-1，心态+1（人类恢复）
- **“又开会？那我吃点零食”**：现金-20，心态+2
- **“写 Finding 写到怀疑人生”**：写作+小概率成长；心态-2
- **“拉群聊：‘这个修复什么时候上？’”**：沟通消耗；修复率↑
- **“看竞赛榜单刷新”**：心态波动（随机 +1 / -2）
- **“给自己立个规矩：不接加急”**：短期订单少，长期漏报率↓

---

## 5. 新的“可落地”数值（不要太多，但要有味道）
建议只加 2~3 个，避免 UI 爆炸：
- **Focus（专注）**：影响高危发现率与误判率
- **Fix Rate（修复率，项目内）**：影响爆雷权重与声望结算
- **Public Heat（舆情热度，可选）**：影响事件触发与声望波动（“出名是风险”）

---

## 6. 让“漏洞”更像“洞”（叙事层面）
现在漏洞只是 S/H/M/L 分值；可以在不做真代码的前提下，加“洞的类型标签”，增强真实感与笑点：
- **权限/升级点**、**精度/舍入**、**重入/回调**、**预言机/操纵**、**跨链消息**、**签名重放**、**MEV 可提取**、**经济攻击**…

在日志里就能写出很像真的一句话：
- “你在 upgradeTo 上闻到熟悉的味道：那种‘一键穿透’的味道。”
- “预言机在边界条件处变得很诚实：它只是不再说真话。”

## 7. 内容扩充的优先级（建议迭代顺序）
### v0.2（最划算的增强：更好玩但改动小）
- 平台：加入提交/冻结/申诉状态
- 直客：加入修复率 + 复测门槛
- 事件：做 3 条事件链（背书/范围/平台撞车）
- 动作：把按钮文案生活化 + 给每周日志更有梗

### v0.3（职业化增强：更像模拟器）
- Focus 资源 + 脑雾/透支链
- 工具链装备与维护成本
- 客户/评审/社区标签系统

### v0.4（长期目标：内容深、但工程量大）
- 团队系统（招人、绩效、离职、泄密）
- 法务与披露流程（责任边界、公告措辞、争议仲裁）
- 爆雷后续（追责/公关/赔偿/复盘）

---

## 8. 事件库模板（便于你后续继续写）
每条事件建议统一结构，方便落地到代码：
- **id**
- **标题**
- **触发条件**
- **文本（可带变量：客户标签/协议类型/热度）**
- **选项 A/B/C**
  - 消耗（行动点/Focus/现金）
  - 立即收益（声望/平台评级/修复率）
  - 延迟后果（下周爆雷权重↑、争议事件解锁、客户关系变化）

---

## 9. 风格边界（避免现实风险但保留讽刺）
- 不影射真实公司/项目/个人；平台规则全部写成“抽象玩法”
- 吐槽针对“现象”不针对“对象”
- 允许黑色幽默，但保留“严肃模式”开关（减少梗、偏职业模拟）
